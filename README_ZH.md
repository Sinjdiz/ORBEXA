## ⚠️ 重要声明

> **我们是基础设施服务商，不是支付中介。**
>
> - 🌐 **用户域名发布**：AI 代理通过商户自己的域名访问，而非 ORBEXA 的域名
> - 💳 **不碰资金**：所有支付在商户原平台完成，我们不经手任何交易资金
> - 🔒 **数据归属**：商品数据归商户所有，我们只做标准化处理和协议转换

## 我们解决什么问题？

现在的 AI 助手很聪明，能聊天、能写代码、能做翻译。但你让它帮你买个东西？对不起，做不到。

问题出在哪？AI 读不懂电商网站的数据，也没有统一的方式去搜索商品、下单、查物流。

**ORBEXA 就是这个桥梁。**

我们把商家的商品数据标准化，用 AI 能理解的方式暴露出去。商家加一条 DNS 记录，AI 就能直接帮用户在他店里买东西。

## 数据处理能力

### 🧹 自动清洗
| 处理 | 说明 |
|------|------|
| 去重 | 相同 SKU 的商品只保留一条 |
| 格式化 | 价格统一到小数点后两位，货币标准化 |
| 分类标准化 | 把 "女装/裙子/连衣裙" 映射到标准分类体系 |
| 库存修正 | 负数库存自动归零，状态同步 |
| 图片优化 | 检测无效图片链接，提示更换 |

### ✨ AI 自动补全
| 字段 | 补全方式 |
|------|----------|
| 商品描述 | 根据标题和图片 AI 生成 |
| SEO 关键词 | 自动提取商品特征词 |
| 分类标签 | 图片识别 + 文本分析 |
| 属性规格 | 从描述中提取颜色、尺码等 |

### 📊 质量评分
每个商品都有一个 0-100 的质量分，影响因素：

| 权重 | 字段 |
|------|------|
| 25% | 标题完整性（字数、关键词） |
| 25% | 描述质量（长度、信息量） |
| 20% | 图片质量（数量、分辨率、有效性） |
| 15% | 价格合理性（跟同类商品对比） |
| 15% | 属性完整性（颜色、尺码、材质等） |

低于 60 分的商品会被标记，提示商户优化。

## 支持的 AI 协议

### UCP (Google Universal Commerce Protocol)

基于 Schema.org 的语义化协议,Google 主推。

```json
// 请求示例
GET /ucp/products?query=红色连衣裙&limit=10

// 响应示例
{
  "@context": "https://schema.org",
  "@type": "ItemList",
  "numberOfItems": 10,
  "itemListElement": [
    {
      "@type": "Product",
      "productID": "SKU001",
      "name": "法式红色碎花连衣裙",
      "offers": {
        "@type": "Offer",
        "price": 299.00,
        "priceCurrency": "CNY",
        "availability": "InStock"
      }
    }
  ]
}
```

### MCP (Anthropic Model Context Protocol)

Claude 原生支持的 JSON-RPC 协议。

```json
// 工具调用示例
{
  "jsonrpc": "2.0",
  "method": "tools/call",
  "params": {
    "name": "search_products",
    "arguments": {
      "query": "红色连衣裙",
      "max_price": 500
    }
  }
}
```

### ACP (Agentic Commerce Protocol)

OpenAI 生态的 REST 风格协议。

```json
// 请求示例
POST /acp/v1/search
{
  "intent": "find_product",
  "query": "红色连衣裙",
  "constraints": {
    "budget": 500,
    "currency": "CNY"
  }
}
```
